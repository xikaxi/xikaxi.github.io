示例请求 Order URL✍

Request:☟

URL https://sellingpartnerapi-na.amazon.com/orders/v0/orders?MarketplaceIds=ATVPDKIKX0DER&CreatedAfter=2020-06-06
Message 返回某时间段开始之后 或者之前的订单
Request GET
URI /orders/v0/orders

Parameter:☟

key value 备注
MarketplaceIds ATVPDKIKX0DER 市场编号 MarketplaceId值的列表。用于选择放置在指定市场中的订单。

Optional parameters:☟

key value 备注
CreatedAfter 2020-12-25 用于选择在指定时间之后（或之后）创建的订单的日期。仅返回指定时间之后下的订单。需要CreatedAfter参数或LastUpdatedAfter参数。两者都不能为空。日期必须为ISO 8601格式。
CreatedBefore 2020-12-25 用于选择在指定时间之前（或指定时间）创建的订单的日期。仅返回指定时间之前下的订单。日期必须为ISO 8601格式。
LastUpdatedAfter 2020-12-25 用于选择在指定时间（或之后）最后更新的订单的日期。更新被定义为订单状态的任何更改，包括创建新订单。包括亚马逊和卖方所做的更新。日期必须为ISO 8601格式。
LastUpdatedBefore 2020-12-25 用于选择在指定时间之前（或指定时间）最后更新的订单的日期。更新被定义为订单状态的任何更改，包括创建新订单。包括亚马逊和卖方所做的更新。日期必须为ISO 8601格式。
OrderStatuses Shipped(已发货) 用于过滤结果的OrderStatus值列表。可能的值：PendingAvailability（此状态仅适用于预购。已下达订单，未授权付款，并且该商品的发布日期在将来。）待处理（已下达订单，但未授权付款）；未发货（付款已被授权且订单已准备好发货，但订单中没有任何物品已发货）；部分发货（订单中的一个或多个（但不是全部）物品已发货）；已发货（订单中的所有物品均已发货）；未确认的发票（订单中的所有物品均已发货。卖方尚未向亚马逊确认已将发票已发货给买方。）；已取消（订单已被取消）；和无法实现（无法完成订单。
